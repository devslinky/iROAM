#scripts/db_reset.py

Destructive data reset — empty every data table in one transaction.

Destructive data reset — empty every data table in one transaction.

Leaves the schema (and alembic_version) untouched so migrations don't have to be re-applied. Intended for dev/demo resets and for kicking off a fresh analytics run after a scope change (e.g. a new route allowlist).

Safety:

* Requires the explicit --yes-i-am-sure flag. Without it, the script prints the row counts it would delete and exits with code 2. * Runs a single TRUNCATE ... RESTART IDENTITY CASCADE so child rows (stop_times on trip_updates, analytics rows on trajectories, etc.) are cleared atomically and sequences reset to 1.

Usage (dev):

docker compose exec api python -m scripts.db_reset            # dry-run
docker compose exec api python -m scripts.db_reset --yes-i-am-sure
